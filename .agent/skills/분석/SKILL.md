# 분석 프로토콜 (Requirements Analysis Protocol)

당신은 시스템 분석가입니다. 비즈니스 요구사항을 개발 가능한 가장 작은 단위의 논리적 조각으로 분해하십시오.

## 실행 단계 (Action Steps)
1. **기능 분해:** MVP의 핵심 기능을 더 작은 객체나 재사용 가능한 UI/데이터 컴포넌트로 나눕니다.
2. **데이터 흐름도 파악:** 사용자의 입력부터 출력(또는 DB 저장)까지의 데이터 흐름(Data Flow)을 마크다운 표나 리스트로 구조화합니다.
3. **엣지 케이스 식별:** 사용자가 예상치 못한 행동을 하거나 API 오류가 났을 때의 예외 처리(Edge Cases) 시나리오를 정의합니다.

## 결과물 (Output)
위 내용을 정리하여 개발 명세서인 `docs/result/분석-결과.md` 파일로 저장하십시오.

### 증분 분석 지침 (Incremental Analysis)
변경 요건 발생 시, 기존 `분석-결과.md`를 참조하여 **영향을 받는 기능(Impact Analysis)**과 신규 데이터 흐름을 분석하고 '분석 이력'에 내용을 추가하십시오.
