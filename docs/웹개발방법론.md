# 1인 기업 웹 개발 표준 (1-Person Enterprise Web Development Standards)

이 문서는 프로젝트의 기획부터 배포까지 전체 소프트웨어 개발 수명 주기(SDLC)에 걸쳐 AI 에이전트가 코드를 생성하거나 구조를 설계할 때 반드시 지켜야 하는 제약 조건입니다. 모든 단계에서 '1인 기업의 생산성'과 '유지보수의 용이성'을 최우선으로 합니다.

---

## Phase 1 & 2: 요구사항 정의 및 분석/설계 (Requirements & Design)
문서 작성 및 시스템 아키텍처 설계 시 다음 원칙을 준수합니다.

- **MVP 중심 설계:** 거창한 기능보다는 고객의 핵심 가치 1가지를 검증하는 최소 기능 제품(MVP) 형태로 기능을 제한하고 기획 문서를 작성합니다.
- **백엔드리스 (Backendless) 우선:** 별도의 백엔드 서버 인프라를 설계하지 않습니다. Next.js 등의 메타 프레임워크 내에서 TypeScript 기반의 서버 함수(Server Functions)를 사용하거나, Firebase/Supabase 같은 BaaS(Backend as a Service)를 활용하여 아키텍처를 구성합니다.

## Phase 3: 구현 (Implementation) - 로직 및 아키텍처
프론트엔드 및 비즈니스 로직 코드를 작성할 때 다음 규칙을 강제합니다.

- **표준 라이브러리 사용:** 데이터 페칭, 캐싱, 상태 동기화 및 폼 관리는 TanStack 제품군(Query, Form 등)을 기본으로 사용합니다.
- **레거시 최적화 지양:** React Compiler가 적용되는 2026년 환경을 전제로 하므로, `useMemo`, `useCallback`, `React.memo` 등의 수동 메모이제이션을 작성하지 말고 직관적인 코드를 작성합니다.
- **코드 문서화:** Airbnb React JSX 스타일 가이드를 준수하며, 복잡한 함수 로직에는 반드시 인라인 주석을 달고 Prop Types나 TypeScript 인터페이스를 명확히 문서화합니다.

## Phase 3: 구현 (Implementation) - UI 및 웹 접근성 (Accessibility & SEO)
퍼블리싱 및 스타일링 코드 생성 시 다음 웹 접근성 및 최적화 규칙을 반드시 지킵니다.

- **하이브리드 스타일링:** 레이아웃과 간격은 Tailwind CSS(유틸리티 클래스)로 처리하고, 브랜드 테마나 컬러는 네이티브 CSS 변수(Custom Properties)를 결합하여 사용합니다.
- **시맨틱 마크업:** 무의미한 `<div>` 남발을 금지하며, 사용 목적에 맞게 `<main>`, `<section>`, `<article>`, `<nav>` 등을 사용합니다. `<section>`이나 `<article>` 내부에는 반드시 Heading 태그를 포함합니다.
- **Heading 계층 준수:** `h1`부터 `h6`까지 중간 단계를 생략하지 않고 순서대로 사용합니다.
- **대체 텍스트(Alt):** 의미 없는 단순 디자인용 이미지에는 빈 alt 값(`alt=""`)을 부여합니다.
- **클릭 이벤트 제한:** 클릭 이벤트는 `<button>`이나 `<a>` 태그에 부여하는 것을 원칙으로 하되, 불가피하게 다른 요소(div 등)에 적용할 경우 반드시 `role="button"`을 명시합니다.
- **포커스 상태 유지:** 키보드 접근성을 위해 CSS로 아웃라인을 강제 삭제하지 않으며, 필요시 `focus-visible`을 활용합니다.
- **상태 값 제공:** 토글이나 활성화 상태는 `aria-expanded` 또는 `aria-current` 속성을 통해 스크린 리더에 상태를 전달합니다.

## Phase 4 & 5: 테스트 및 배포 (Testing & Production)
- **보안 기본값 (Security by Default):** 프론트엔드가 백엔드 역할까지 담당하므로, 데이터 접근이나 서버 함수 작성 시 권한 검증(Auth/Validation) 로직을 필수로 포함하여 취약점을 방지합니다.
- **엣지 배포 (Edge Deployment) 호환성:** 복잡한 인프라 관리 없이 자동 스케일링이 가능하도록, 생성되는 모든 코드는 Vercel 등의 엣지 런타임 환경에서 동작할 수 있도록 설계합니다.
