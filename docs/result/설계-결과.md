## 1. 백엔드리스 아키텍처 (Backendless Design)

### 1.1 인프라 스택
- **Hosting/Edge:** Cloudflare Pages
- **Database:** Cloudflare D1 (SQLite 기반 서버리스 DB)
- **Functions:** Cloudflare Pages Functions (Edge Runtime)

### 1.2 DB 스키마 설계 (D1)
```sql
CREATE TABLE licenses (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    customer_email TEXT NOT NULL,
    license_key TEXT NOT NULL UNIQUE,
    plan_type TEXT DEFAULT 'Pro',
    status TEXT DEFAULT 'active',
    issued_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    last_verified_at TIMESTAMP
);

CREATE TABLE verification_logs (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    license_id INTEGER,
    attempted_email TEXT,
    ip_address TEXT,
    status TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY(license_id) REFERENCES licenses(id)
);
```

## 2. 상태 관리 및 로직 설계 (State & Logic)

### 2.1 서버 상태 관리 (TanStack Query)
- **Query Keys:** 
  - `['licenses']`: 라이선스 목록 및 검색 결과
  - `['license', id]`: 개별 라이선스 상세 정보
- **Mutations:**
  - `useMutation(createLicense)`: 신규 라이선스 발급 및 리스트 무효화(`invalidateQueries`)
  - `useMutation(updateStatus)`: 라이선스 활성화/비활성화 전환

### 2.2 클라이언트 사이드 데이터 흐름
1. `Admin.tsx` 접속 시 `useQuery(['licenses'])` 실행.
2. 데이터 캐싱 및 로딩 UI 노출.
3. 라이선스 생성 버튼 클릭 시 `useMutation` 호출 -> API 서버 전송 -> 성공 시 캐시 갱신.

## 3. UI 컴포넌트 계층 설계 (UI Framework)

### 3.1 컴포넌트 계층 (Component Hierarchy)
- **Layouts:** `BrowserRouter` -> `QueryClientProvider` -> `MainLayout`
- **Pages:**
  - `LandingPage`: Hero, Demo(Markdown), Features, Pricing, Roadmap
  - `AdminPage`: Sidebar, StatsCards, LicenseTable, ModalForm
- **Shared UI:** `Button`, `Input`, `Badge`, `CopyButton` (Lucide Icons 연동)

### 3.2 디자인 시스템 (Hybrid Styling)
- **Layout/Spacing:** Tailwind CSS (정형화된 간격 유지)
- **Colors/Brands:** CSS Variables (`--blue-600`, `--slate-900` 등)를 통해 테마 관리.
- **Micro-animations:** `motion/react`를 활용한 페이지 전환 및 리스트 인터랙션.
