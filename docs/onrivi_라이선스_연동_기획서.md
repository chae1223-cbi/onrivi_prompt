# OnriviPrompt 라이선스(License) 발급 및 인증 흐름 기획서

성공적인 소프트웨어 판매를 위해 필수적인 **결제 연동 - 라이선스 키 자동 발급 - 앱 내 실시간 인증**의 전체 워크플로우를 기획한 문서입니다.

## 1. 개요 및 요건
* 목표: 고객이 랜딩페이지에서 결제를 완료하면 1초 내로 라이선스 키가 발급되고, 데스크탑 앱(OnriviPrompt)에서 이 키를 입력하여 "Pro" 모드를 활성화할 수 있어야 합니다.
* 비용 무과금 지향: 월정액 고정 비용을 내는 서버 대신(비용 최소화), 사용한 만큼만 내는 Serverless(서버리스) 아키텍처를 추천합니다.

---

## 2. 권장 기술 스택 연동 아키텍처 

`결제 위젯(Paddle) -> 서버리스 (Cloudflare/Supabase) -> 이메일 발송 -> 유저가 앱에 입력` 구조입니다.

### [Phase 1: 결제 및 Key 발급 (백엔드)]
1. **결제 모듈 (Paddle / FastSpring)**
   - 랜딩 페이지 메인 UI의 "Buy License" 버튼을 클릭 시 결제 팝업(Checkout Widget)이 뜹니다.
   - 플랫폼이 세금(VAT)까지 알아서 계산해 카드로 결제받습니다.
2. **웹훅 수신 서버 (Cloudflare Workers)**
   - 결제가 성공하면 승인 알림(Webhook)이 우리 서버로 날아옵니다.
   - Node.js 스크립트가 32자리 난수의 `고유 라이선스 키(GUID)`를 하나 생성합니다.
3. **DB 저장 (Supabase / Firebase)**
   - `Licenses` 데이터베이스 테이블에 [고객 이메일, 생성된 키, 발급일시, 사용 상태]를 기록합니다.
4. **발급 알림 (Brevo / SendGrid API)**
   - 등록된 고객 이메일로 "OnriviPrompt 라이선스 키 발급 안내" 메일을 자동 발송합니다. (다운로드 링크 포함)

### [Phase 2: 앱 내 활성화 (프론트/데스크탑)]
1. **유저 키 입력**
   - 유저가 OnriviPrompt 앱의 "설정(위젯)"에 들어가 이메일로 받은 라이선스 키를 붙여넣기 합니다.
2. **온라인 검증 API 호출**
   - 앱이 내부적으로 `https://api.onrivi.com/verify-license` 로 키를 보냅니다.
3. **DB 상태 일치 확인 및 로컬 승인**
   - 서버에서 DB를 조회해 "합법적인 활성 키"가 맞다면 OK 사인을 보내며, 앱 내부 LocalStorage/Config에 "Pro 모드 활성화" 플래그가 저장되어 무제한 사용 및 이미지 생성이 풀리게 됩니다.

## 3. 대표님 결정 필요 (다음 스텝)
이 흐름을 실제로 구축하기 위해서는 다음 플랫폼의 가입 및 API 키 발급이 필요합니다.

* [ ] 결제 대행 (MoR): **Paddle** 혹은 **Lemon Squeezy** 개발자 계정 가입
* [ ] DB 저장소: **Supabase** (가장 추천) 무료 계정 가입
* [ ] 이메일 발송: **Brevo** 무료 계정 가입 (월 9,000건 무료)

이 3가지만 대표님 이메일로 무료가입해 주시면, 제가 위 워크플로우가 자동으로 돌아가는 API 서버(Cloudflare) 코드를 짜서 연동해 드릴 수 있습니다!
