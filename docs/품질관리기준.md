# 1인 기업 MVP 품질 관리 기준 (Quality Standards for 1-Person MVP)

이 문서는 1인 기업 환경에서의 빠른 MVP 개발과 안정적인 유지보수를 위한 핵심 품질 지표를 정의합니다. 정기적인 품질 검토와 시스템의 건전성을 유지하는 데 목적이 있습니다.

---

## 1. 핵심 품질 지표 (Key Metrics)

| 영역 | 관리 항목 | 준수 목표 |
| :--- | :--- | :--- |
| **보안** | API 키/환경 변수 노출 방지 | 100% (결함 불허) |
| **코드** | TanStack Query/Form 표준 준수 | 95% 이상 |
| **접근성** | 시맨틱 HTML 및 Heading 계층 | 필수 준수 |

## 2. 영역별 상세 검증 기준

### 2.1 보안 및 가드레일 (Security & Guardrail)
- **환경 변수 관리:** 클라이언트 측(`process.env.NEXT_PUBLIC_` 등)에 보안상 민감한 정보가 노출되지 않도록 서버 함수 내부에서만 비밀값 처리.
- **권한 검증:** 모든 서버 함수(Server Functions/Actions)는 호출 시 유효한 인증 및 권한 확인 로직을 포함.

### 2.2 코드 아키텍처 (Code Architecture)
- **표준 라이브러리:** 복잡한 상태 관리 대신 TanStack Query를 활용하여 서버 상태를 관리하고, 캐싱 정책을 명확히 설정.
- **모듈화:** 하나의 컴포넌트나 함수가 200라인을 넘지 않도록 작게 유지하며, 공통 로직은 `utils` 또는 전용 `hooks`로 분리.

### 2.3 UI/UX 및 접근성 (A11y & SEO)
- **시맨틱 태그:** `<main>`, `<nav>`, `<header>`, `<footer>`, `<section>` 등을 적재적소에 사용.
- **키보드 접근성:** 모든 인터랙티브 요소는 키보드로 접근 가능해야 하며, `focus-visible` 상태에서 시각적 피드백이 명확해야 함.
- **반응형 디자인:** 모바일 우선 디자인(Mobile-First)을 기본으로 하며, 다양한 해상도에서 레이아웃 깨짐이 없어야 함.

### 2.4 성능 및 배포 가능성 (Performance & Deployability)
- **메모이제이션:** React Compiler 도입을 고려하여 `useMemo`, `useCallback` 등을 남발하지 않고, 렌더링 최적화는 컴포넌트 구조 설계를 통해 해결.
- **엣지 호환성:** Node.js 특정 라이브러리 사용을 지양하고, 표준 Web API를 사용하여 어디서든 배포 가능한 상태 유지.
