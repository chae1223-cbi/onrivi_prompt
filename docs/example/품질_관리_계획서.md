# 뉴뉴스메일(NewNewsMail) 품질 관리 계획서 v1.0

## 1. 개요
본 문서는 뉴뉴스메일 서비스의 신뢰성과 지속 가능성을 보장하기 위한 품질 관리 표준을 정의합니다. AI 에이전트가 생성하는 콘텐츠의 정확성, 사용자의 법적 권리 보호, 그리고 시스템의 안정적인 운영을 핵심 품질 목표로 설정합니다.

---

## 2. 품질 관리 지표 (KPI)

| 관리 영역 | 핵심 품질 지표 (KPI) | 목표 수준 |
| :--- | :--- | :--- |
| **AI 콘텐츠** | 요약의 '결론 우선' 원칙 준수율 | 95% 이상 |
| **비주얼** | 이미지 내 텍스트 미발생 및 합성 가독성 | 100% 준수 |
| **법적 준수** | `ConsentLogs` 기록 누락률 | 0% (결함 불허) |
| **시스템** | 이메일 발송 성공률 (Delivery Rate) | 99% 이상 |
| **유지보수** | 코드 중복률 최적화 및 공통 모듈 활용 | 상시 개선 |
| **사용자 경험** | 에이전트 상태 모니터링 정확도 | 실시간 동기화 |

## 3. 영역별 품질 검증 절차

### 3.1 AI 요약 및 번역 품질 (AI Quality)
- **프롬프트 준수 검사**: 제미니가 생성한 JSON 결과물이 [프롬프트 설계서](./프롬프트_설계서.md)에 정의된 스키마를 준수하는지 자동 검증.
- **다국어 무결성**: 수신자의 `PREF_LANG`에 따른 번역 결과의 자연스러움 및 오역 여부 샘플링 검수.
- **중복 검출**: 동일한 또는 유사한 내용의 기사가 중복 발송되지 않도록 URL 및 제목 기반 사전 필터링 강화.

### 3.2 비주얼 에셋 품질 (Visual Quality)
- **이미지 정합성**: 생성된 배경 이미지가 뉴스 기사의 카테고리와 시각적으로 연관성이 있는지 확인.
- **합성 텍스트 가독성**: 배경색과 합성된 제목 텍스트 간의 대비(Contrast)를 확보하여 가독성 보장.
- **파일 정제**: 파손된 이미지 파일이나 로딩 실패 에셋에 대한 자동 예외 처리 및 대체 이미지(Alt) 적용.

### 3.3 법적 준수 가드레일 (Compliance Quality)
- **로그 정합성**: 구독 신청 및 철회 시 `ConsentLogs`에 `IP`, `Timestamp`, `ActionType`이 즉시 기록되는지 주기적 전수 조사.
- **수신거부 즉시성**: 로그인 없는 원클릭 수신 거부 기능이 3초 이내에 완료되고 반영되는지 기술적 테스트 수행.
- **광고 표기 준수**: 제목의 `(광고)` 접두어 누락 여부를 발송 전 시스템적으로 강제 검증.

### 3.4 코드 및 유지보수 품질 (Maintainability Quality)
- **공통 모듈 준수**: `BaseAgent` 상속을 강제하고, DB 연결 등 공통 기능을 일원화하여 중복 소스를 최소화함.
- **자동 품질 감사**: `scripts/quality_guard.py`를 통해 코드 표준, 미사용 파일, DB 무결성을 상시 점검함.
- **UI/UX 표준 준수**: 모든 화면 개발 시 `docs/화면_제작_가이드.md`에 정의된 디자인 토큰 및 레이아웃을 준수함.

### 3.5 테스트 격리 및 운영 반영 정책 (Test Isolation Policy)

> [!IMPORTANT]
> **`testprogram/` 폴더는 순수 테스트 전용 공간입니다. 운영 코드에서 `testprogram/`의 파일을 직접 참조하는 것을 금지합니다.**

#### 개발 워크플로우

```
[1단계] testprogram/ 에서 아이디어 검증 및 테스트
      ↓ 테스트 통과
[2단계] 로직을 운영 위치로 이전 및 정식 작성
      - Python 스크립트 → scripts/
      - API 라우트    → dashboard/app/api/
      ↓
[3단계] 운영 코드에서 해당 파일 참조
      ↓
[4단계] quality_guard.py 로 testprogram 참조 여부 자동 검증
```

#### 운영 위치 기준표

| 파일 유형 | 운영 위치 | 테스트 위치 |
|-----------|-----------|-------------|
| Python 에이전트/스크립트 | `scripts/` | `testprogram/` |
| Next.js API 라우트 | `dashboard/app/api/` | `testprogram/` |
| 설정 파일 | `config/` | `testprogram/` |
| DB 마이그레이션 | `scripts/migrations/` | `testprogram/` |

#### 운영 반영 체크리스트
- [ ] `testprogram/`의 로직이 `scripts/` 또는 해당 위치에 정식 파일로 작성되었는가?
- [ ] 운영 코드가 `testprogram/`을 직접 참조하지 않는가? (`quality_guard.py` 자동 검사)
- [ ] 정식 파일이 `BaseAgent`를 상속하거나 `.env` 로드 방식을 준수하는가?
- [ ] 테스트 파일(`testprogram/`)은 정리(삭제 또는 유지) 여부를 결정했는가?

## 4. 상시 품질 감시 시스템 (Automated Quality Guard)
시스템의 지속적인 건전성을 위해 상시 자동 Audit 체계를 가동합니다.

1. **Agent Standard**: 모든 에이전트의 `BaseAgent` 상속 여부 및 하드코딩된 연결 지점 탐사.
2. **Resource Hygiene**: 가이드라인(testprogram 격리 등)을 벗어난 임시 파일 및 중복 파일 잔존 여부 감시.
3. **Docs Sync**: 핵심 사양서(`docs/specs/*.md`) 및 지침서(`화면_제작_가이드.md`)의 존재 여부 확인.
4. **Audit Report**: 점검 결과를 [품질_감사_리포트.md](./품질_감사_리포트.md)로 자동 발행.
