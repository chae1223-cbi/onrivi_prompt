# 화면 제작 가이드 (UI/UX Development Guide) v1.0

본 가이드는 뉴뉴스메일(NewNewsMail) 대시보드의 시각적 일관성과 사용자 경험(UX) 품질을 유지하기 위한 표준 정의서입니다. 모든 신규 화면 및 컴포넌트 개발 시 본 가이드를 준수해야 합니다.

---

## 1. 기본 레이아웃 (Base Layout)

### 1.1 전체 스켈레톤 (Skeleton)
- **사이드바 (Sidebar)**: 너비 `260px`, 고정 좌측 배치. 배경색 `#ffffff` 또는 매우 연한 그레이.
- **헤더 (Header)**: 높이 `64px`, 상단 고정. 현재 페이지 명칭 및 사용자 프로필 배치.
- **콘텐츠 영역 (Main Content)**: 헤더와 사이드바를 제외한 나머지 영역. 배경색 `#f8fafc` (연한 블루 그레이).

### 1.2 간격 및 정렬 (Spacing & Grid)
- **페이지 패딩**: 콘텐츠 영역 전체에 `32px (p-8)` 기본 적용.
- **그리드 시스템**: 8px 단위 리듬 (`padding: 8, 16, 24, 32 / margin: 8, 16, 24, 32`).
- **섹션 간격**: 카드 형태의 섹션 사이 간격은 `24px (gap-6)` 적용.

---

## 2. 디자인 토큰 (Design Tokens)

### 2.1 컬러 팔레트 (Colors)
- **Primary (Brand)**: `#0284c7` (Sky Blue) - 버튼, 하이라이트, 링크.
- **Success**: `#10b981` (Emerald) - 완료, 정상 상태.
- **Warning**: `#f59e0b` (Amber) - 대기, 주의.
- **Danger**: `#ef4444` (Red) - 오류, 삭제, 경고.
- **Background**: `#f8fafc` (Main), `#ffffff` (Card/Sidebar).
- **Text**: `#0f172a` (Headline), `#475569` (Body), `#94a3b8` (Muted).

### 2.2 타이포그래피 (Typography)
- **기본 폰트**: Pretentard, Inter 또는 Roboto (Sans-serif 계열).
- **H1 (Page Title)**: `28px`, Bold, `#0f172a`.
- **H2 (Section Header)**: `20px`, SemiBold, `#1e293b`.
- **Body**: `14px`, Medium, `#475569`.

---

## 3. 표준 컨트롤 및 컴포넌트 (Components)

### 3.1 버튼 (Buttons)
- **높이**: 기본 `42px`, 작은 버튼 `32px`.
- **반경 (Border Radius)**: `10px` (둥근 사각형).
- **스타일**:
  - Primary: Filled (`#0284c7`), White Text.
  - Secondary: Ghost 또는 Light Blue Background.
  - Hover: 투명도 조절 또는 색상 10% 어둡게.

### 3.1.1 버튼 배치 (Button Placement)
- **우측 최우선 배치**: '취소', '닫기', 'Cancel', 'Close' 등 작업을 중단하거나 창을 닫는 성격의 버튼은 버튼 그룹(Button Group) 내에서 **항상 가장 우측**에 배치합니다.
- **예시**: `[저장] [실행] [취소]` (O) / `[취소] [저장] [실행]` (X)

### 3.2 입력 필드 (Inputs)
- **Label**: 입력창 상단에 고정 표시 (Floating 보다는 고정형 선호).
- **Border**: `1px solid #e2e8f0`. Focus 시 `#0284c7`로 변경.
- **Padding**: `12px 16px`.

### 3.3 데이터 테이블 (Tables)
- **Header**: 배경색 `#f1f5f9`, 텍스트 Bold, 폰트 크기 `13px`.
- **Row**: 마우스 호버(Hover) 시 `#f8fafc` 적용.
- **Cell Content**: 세로 중앙 정렬 기본.

## 4. 품질 관리 체크리스트 (UI QA)
- [ ] 반응형 대응 (모바일/태블릿 가독성 확보) 여부.
