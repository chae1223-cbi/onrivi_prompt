# 뉴뉴스메일(NewNewsMail) 기능 설계서 v1.0

## 1. 개요
본 문서는 뉴뉴스메일 서비스의 핵심 기능 범위와 각 기능별 작동 규칙을 정의합니다. 인공지능 에이전트가 수행해야 할 업무 단위를 구체화하여 시스템 구현의 기준점으로 활용합니다.

---

## 2. 핵심 기능 명세

### 2.1 AI 콘텐츠 엔진 (Intelligence)
| 기능 ID | 기능명 | 주요 설명 및 작동 규칙 |
| :--- | :--- | :--- |
| **F-CNT-01** | 뉴스 수집 | `IntakeAgent`를 통한 RSS 및 Google News 수집 (요일별 전략 기반) |
| **F-CNT-02** | AI 요약 | `SummaryAgent`를 통한 [Brief, Basic, Detail] 요약 및 인사이트 도출 |
| **F-CNT-03** | 이미지 생성 | `VisualAgent`를 통한 배경 이미지 획득 및 타이포그래피 합성 |

### 2.2 운영 및 품질 관리 (Operations)
| 기능 ID | 기능명 | 주요 설명 및 작동 규칙 |
| :--- | :--- | :--- |
| **F-OPS-01** | 스케줄링 | `SchedulerAgent`를 통한 주기적 자동 파이프라인(수집-요약-이미지-발송) 구동 |
| **F-OPS-02** | 품질 감시 | `QualityGuard`를 통한 소스 코드 표준 및 UI 가이드 준수 여부 상시 Audit |
| **F-OPS-03** | 프로젝트 백업 | `Backup Utility`를 통한 일 단위 물리적 백업본(.zip) 생성 |

### 2.3 법적 준수 및 가드레일 (Compliance)
| 기능 ID | 기능명 | 주요 설명 및 작동 규칙 |
| :--- | :--- | :--- |
| **F-CMP-01** | 규정 준수 발송 | 제목 앞 `(광고)` 표기 및 푸터 전송자 정보 강제 삽입 |
| **F-CMP-02** | 원클릭 거부 | 로그인 없는 즉시 수신 거부(No-Login Unsubscribe) 및 이력 관리 |
| **F-CMP-03** | 정기 동의 확인 | 마지막 동의일 기준 2년 주기 수신 동의 확인 메일 자동 발송 |

### 2.4 관리 및 모니터링 (Admin)
| 기능 ID | 기능명 | 주요 설명 및 작동 규칙 |
| :--- | :--- | :--- |
| **F-ADM-01** | 에이전트 대시보드 | 수집/요약/발송 에이전트 가동 현황 및 API 사용량 모니터링 |
| **F-ADM-02** | 콘텐츠 검수 | AI가 생성한 요약본 및 이미지의 최종 검토 및 수정 권한 |
| **F-ADM-03** | 성과 분석 | 오픈율, 클릭률 등 이력 데이터를 기반으로 한 서비스 통계 제공 |

---

## 3. 운영 원칙
1. **로그 우선**: 모든 중요 액션은 `CONSENT_LOGS` 또는 `EMAIL_SEND_LOGS`에 즉시 기록되어야 함.
2. **AI 면책**: 모든 생성 콘텐츠 결과물에 대해 AI 생성 정보의 오차 가능성을 명시함.
3. **규격 준수**: 모든 데이터 교환은 정의된 API 명세를 준수함.
