# 뉴뉴스메일(NewNewsMail) 화면 설계서 v1.0

## 1. 개요
본 문서는 뉴뉴스메일 서비스의 UI/UX 설계를 정의합니다. 기술적 복잡도를 최소화하고 정보의 가독성과 법적 준수 가이드라인을 최우선으로 고려합니다.

---

## 2. 표준 화면 레이아웃 (Layout Strategy)
시스템 전반에 **Tailwind CSS**를 활용한 일관된 디자인 시스템을 적용합니다.
1. **Header**: 서비스 플랫폼 로고 및 사용자 프로필/설정 접근.
2. **Main Content**: 서비스의 핵심 기능을 담은 중앙 영역.
3. **Footer**: 법적 고지사항, 이용약관, 수신거부 안내 포함.

---

## 3. 화면별 설계 명세

| 화면 ID | 화면명 | 주요 구성 및 목적 |
| :--- | :--- | :--- |
| **SCR-01** | 랜딩 및 가입 | 슬로건 노출, 구독 폼, 법적 동의 체크박스(필수) |
| **SCR-02** | 뉴스레터 상세 | 개인화 키워드 관리, 발송 시간 설정, 요약 수준 선택 |
| **SCR-03** | 구독/결제 | 요금제 비교(Free/Pro), 토스페이먼츠 연동 |
| **SCR-04** | 수신 거부 | 로그인 없는 즉시 수신 거부 처리 및 결과 안내 |
| **ADM-01** | 통합 대시보드 | 에이전트 가동 현황 모니터링 및 발송 통계 |
| **ADM-02** | 법적 로그 관리 | `ConsentLogs` 조회 및 이메일 기반 증빙 검색 |

---

## 4. 주요 UI 포인트

### 4.1 법적 동의 체크박스 (Compliance UI)
- 가입 시 '이용약관' 및 '광고성 정보 수신 동의'를 각각 별도 체크하도록 설계.
- 동의 시점의 IP와 시간을 `ConsentLogs`에 즉시 기록하는 백그라운드 로직 연계.

### 4.2 에이전트 대시보드 (Admin UI)
- 수집/요약/발송 에이전트의 상태를 직관적인 상태 지표(Indicator)로 표시.
- 오류 발생 시 관리자에게 시각적 경고 제공.

---

## 5. UI 가이드라인
- **Color Palette**: 
  - Background: `#FFFFFF` (Pure White)
  - Point Color: `#3182F6` (Toss Blue Style) / `#111827` (Deep Navy)
- **Typography**: 
  - Sans-serif 계열 (Pretendard 추천)
  - 본문 가독성을 위해 넓은 자간 및 행간 유지
- **Interaction**: 
  - 중요한 법적 액션 시 토스트(Toast) 알림 제공
  - 버튼 비활성 상태 시 명확한 시각적 피드백 (`cursor-not-allowed`)
