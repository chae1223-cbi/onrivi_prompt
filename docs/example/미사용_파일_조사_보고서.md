# 미사용 및 임시 파일 조사 보고서 v1.0

본 보고서는 뉴뉴스메일 소스 코드 내에서 더 이상 사용되지 않거나, 개발 및 마이그레이션 과정에서 임시로 생성되어 소스 분량을 차지하고 있는 파일들을 분석한 결과입니다.

---

## 1. `scripts/` 디렉토리 (임시/장애 대응용)

뉴뉴스메일의 핵심 로직은 `intake`, `summary`, `visual`, `dispatch` 에이전트와 이들을 관리하는 `scheduler_agent`에 집중되어 있습니다. 그 외 다수의 파일은 일회성 작업용으로 판단됩니다.

### 1.1 일회성 마이그레이션 및 초기화 스크립트
다음 스크립트들은 DB 스키마 변경이나 초기 데이터 생성 시 사용되었으며, 현재 운영에는 불필요한 것으로 보입니다.
- `db_migration_...` (JS, PY 파일군)
- `fix_db_schema.py`, `update_jobs_schema.py`
- `migrate_...` (상세 페이즈별 마이그레이션 스크립트 전량)
- `init_db.py`, `seed_categories.py` (시스템 초기 구축 시에만 필요)

### 1.2 임시 점검 및 디버깅 스크립트
- `check_db.py`, `check_db_schema.py`, `check_db_temp.py`, `check_results.py`
- `db_inspect.py`, `db_inspect_counts.py`
- `verify_10step_schema.py`, `verify_phase5.py`
- `test_gemini.py`, `find_best_model.py`, `list_models.py`
- `debug_email_delivery.py` (필요 시에만 사용)

### 1.3 중복 또는 레거시 에이전트
- `toc_agent.py`: 현재 `newsletter_agent.py` 내에 기능이 통합되어 있어 단독 참조가 없음.
- `common_codes.py`: 정의된 Enum과 상수가 실제 실행 코드(scripts 내 에이전트들)에서 import되어 사용되지 않음.
- `cleanup_honors.py`: 특정 비즈니스 로직(명예의 전당 등)을 위한 임시 처리용으로 보임.

---

## 2. `dashboard/` (개발 전용 로직)

### 2.1 개발 전용 API 라우트
- `api/auth/login-dev`: 개발 환경에서의 편의를 위한 로그인 라우트로, 운영 환경 배포 시 제거 권장.
- `api/debug/categories`: 카테고리 디버깅용 API.

### 2.2 스타일 및 리소스
- `globals.css`: 현재 19KB로, 프로젝트 규모 대비 상당히 큼. 리디자인 과정에서 사용되지 않게 된 레거시 스타일(Tailwind 관련 등)이 포함되어 있을 가능성이 높음.

---

## 3. 정리 및 권장 조치

### 3.1 정리 대상 (정리 권장)
- `scripts/` 내의 `check_`, `verify_`, `migrate_` 접두어가 붙은 모든 파일.
- 사용되지 않는 레거시 에이전트 (`toc_agent.py`, `common_codes.py` 등).

### 3.2 관리 방안
- **Backup**: 삭제 전 `docs/backup/scripts` 등으로 이동하여 보관 후, 일정 기간 문제가 없으면 영구 삭제.
- **Standardization**: `init_db.py` 등은 필요 시 `scripts/setup/` 하위로 이동하여 관리 포인트 일원화.
