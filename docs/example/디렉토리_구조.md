# 뉴뉴스메일(NewNewsMail) 디렉토리 구조 설명서

최적화 작업이 완료된 프로젝트의 물리적 구조와 각 디렉토리의 역할을 설명합니다.

---

## 📂 전체 구조 (Root Overview)

```text
NewNewsMail/
├── .agent/             # AI 에이전트 전용 지침 및 워크플로우 (기술 스택, 스킬 정의)
├── backups/            # 프로젝트 백업 및 레거시 소스 보관함
├── dashboard/          # Next.js 기반 관리 대시보드 (testprogram으로 임시 파일 격리)
├── db/                 # SQLite 데이터베이스 파일 보관
├── docs/               # 설계서, 사양서, 품질 리포트 등 문서 산출물
├── scripts/            # AI 코어 에이전트 및 운영 자동화 스크립트
├── testprogram/        # 임시 점검용 및 테스트 스크립트 보관함 (품질 관리 제외)
├── .env                # API 키 및 DB 경로 등 환경 변수 설정
├── .gitignore          # Git 관리 제외 설정
└── package.json        # 대시보드 및 전체 프로젝트 의존성 관리
```

---

## 📁 주요 디렉토리 상세

### 1. `scripts/` (AI Core & Utils)
시스템의 실제 비즈니스 로직이 작동하는 곳입니다.
- `base_agent.py`: 모든 에이전트의 공통 기능을 담은 부모 클래스.
- `intake_agent.py`, `summary_agent.py` 등: 실제 AI 작업을 수행하는 에이전트군.
- `scheduler_agent.py`: 정의된 주기에 따라 에이전트들을 가동시키는 엔진.
- `quality_guard.py`: 프로젝트 품질을 상시 점검하는 도구.
- `utils.py`: 시간 계산, DB 상태 업데이트 등 범용 유틸리티 함수.

### 2. `dashboard/` (Frontend & API)
웹을 통해 사용자 및 관리자와 소통하는 영역입니다.
- `app/`, `components/`, `lib/` 등: Next.js 프론트엔드 핵심 소스.
- `testprogram/`: 대시보드 개발 시 사용된 임시 스크립트 및 마이그레이션 도구. (루트와 동일하게 정리됨)
- `public/`: 로고, 생성된 뉴스 이미지 등 정적 자원 보관.

### 3. `docs/` (Documentation & Specs)
프로젝트의 설계와 운영 지침이 담긴 문서 창고입니다.
- `specs/`: 각 프로그램별 상세 동작을 기술한 한글 사양서 보관함.
- `화면_제작_가이드.md`: 대시보드 UI/UX 개발 표준 지침서.
- `품질_관리_계획서.md`: 시스템 유지보수를 위한 품질 기준 정의.
- `프로그램_목록.md`: 전체 파일 인벤토리.

### 4. `testprogram/` (Testing & Temp)
- 임시 점검용, 마이그레이션, 개별 API 테스트 스크립트들이 모여 있습니다.
- 이 폴더 내의 파일들은 프로젝트 상시 품질 관리(`QualityGuard`) 대상에서 제외됩니다.

### 5. `backups/` (Safety Net)
- `scripts_legacy/`: 최적화 과정에서 정리된 미사용 소스 코드를 안전하게 보관.
- `NewNewsMail_backup_*.zip`: `backup_project.py` 실행 시 생성되는 스냅샷 압축 파일.
