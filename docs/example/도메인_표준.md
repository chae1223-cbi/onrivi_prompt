# 뉴뉴스메일(NewNewsMail) 도메인 표준 및 공통 코드 정의서 v1.0

## 1. 개요
본 문서는 시스템 전반에서 사용되는 데이터 타입의 표준(도메인)과 공통 코드 정의를 관리합니다. 일관된 데이터 명명 규칙과 코드 체계를 통해 시스템의 유지보수성을 극대화합니다.

---

## 2. 데이터 도메인 표준

| 도메인 그룹 | 데이터 상세 | 설명 |
| :--- | :--- | :--- |
| **ID** | `INTEGER` | 각 테이블의 고유 식별자 (Auto Increment) |
| **NAME** | `VARCHAR(100)` | 사용자 명칭, 테이블 논리명 등 |
| **EMAIL** | `VARCHAR(255)` | 이메일 주소 (Unique 제약 필수) |
| **TEXT** | `TEXT` | 대용량 텍스트 (기사 원문, 요약본 등) |
| **DATE** | `DATETIME` | 생성/수정/발송 일시 |
| **FLAG** | `BOOLEAN` | 여부 확인 (0: False, 1: True) |

---

## 3. 공통 코드 정의 (`COMMON_CODES`)

| 그룹 코드 | 코드값 | 코드 명칭 | 설명 |
| :--- | :--- | :--- | :--- |
| **PLAN_TYPE** | `FREE` | 무료 플랜 | 기본 요약 및 제한된 발송 |
| | `PRO` | 유료 플랜 | 전체 요약 및 개인화 서비스 제공 |
| **SUB_STATUS**| `ACTIVE` | 활성 | 뉴스레터 정상 발송 상태 |
| | `PAUSED` | 정지 | 사용자 요청에 의한 발송 일시 중단 |
| **LOG_TYPE** | `OPT_IN` | 수신 동의 | 마케팅 정보 수신 동의 시점 로그 |
| | `OPT_OUT` | 수신 거부 | 마케팅 정보 수신 거부 시점 로그 |

---

## 4. 활용 가이이드라인
1. **코드 참조**: 하드코딩을 지양하고 `COMMON_CODES` 테이블에 정의된 `CODE_VALUE`를 사용하여 비즈니스 로직을 구현합니다.
2. **확장성**: 새로운 상태값이 필요한 경우 그룹 코드를 추가하고 하위 코드값을 등록하여 서비스 중단 없이 반영합니다.
3. **AI 통합**: 에이전트는 본 문서에 정의된 도메인 표준을 준수하여 SQL 및 DTO를 생성해야 합니다.
