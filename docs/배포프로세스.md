# 통합 배포 및 Preview 활용 가이드 🚀🌐
Onrivi Prompt 웹 프로젝트의 안전한 개발과 배포를 위한 전체 워크플로우 가이드입니다.

## 1. 개발 및 검증 단계 (`dev02` 브랜치)
새로운 기능 개발이나 수정은 반드시 **`dev02`** 브랜치에서 진행합니다.

### 🌿 코드 수정 후 푸시:
```bash
git add .
git commit -m "feat: 새로운 기능 추가"
git push origin dev02
```

### 🔍 Preview URL 확인:
푸시 즉시 Cloudflare Pages가 Preview 배포를 시작합니다.
**Cloudflare 대시보드 -> Pages -> 프로젝트 선택 -> '실시간 미리보기'** 탭에서 전용 주소(예: `dev02.onrivi-prompt.pages.dev`)를 확인하세요.

## 2. Preview 보안 설정 (GitHub 인증)
테스트 주소가 외부에 노출되지 않도록 GitHub 소셜 로그인을 연동합니다.

- **설정 위치**: Cloudflare Zero Trust -> Access -> Applications
- **방법**: `dev02` 전용 URL에 Policy를 추가하고, Selected Group 또는 Emails에 본인의 GitHub 계정/이메일만 등록합니다.
- **결과**: 이제 해당 URL 접속 시 GitHub 로그인 창이 뜨며, 본인 외에는 누구도 화면을 볼 수 없습니다.

## 3. 운영 배포 단계 (`main` 브랜치)
Preview에서 검증이 완벽히 끝났다면, 실제 사용자용 서버에 반영합니다.

### 🔗 메인 브랜치 병합:
```bash
git checkout main
git merge dev02
git push origin main
```

### 🚀 수동 배포 승인 (GitHub Actions):
우리는 자동 배포를 껐기 때문에 수동으로 실행해야 합니다.
**GitHub 저장소 -> [Actions] 탭 -> 'Deploy Onrivi Prompt' 선택 -> [Run workflow] 클릭!**

### 🗄️ D1 데이터베이스 마이그레이션 (필요 시):
테이블 구조가 바뀌었다면 운영 DB에도 적용해야 합니다.
```bash
npx wrangler d1 migrations apply onrivi-prompt-db --remote
```
