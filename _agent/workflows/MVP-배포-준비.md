---

1. **컴포넌트 리팩터링:** 크기가 너무 크거나 복잡한 컴포넌트(`*.tsx`, `*.jsx`)가 있다면, 작고 재사용 가능한 단위로 분리(Split)하세요. 모든 코드에 Airbnb React JSX 스타일 가이드를 적용하여 일관성을 맞추세요.
2. **웹 접근성 및 SEO 점검:** 화면 마크업이 `main`, `section`, `article` 등의 시맨틱 태그로 구성되었는지 점검하세요. 의미 없는 디자인용 이미지의 `alt` 속성은 빈 값(`alt=""`)으로 처리하고, 클릭 요소가 아닌 곳에 클릭 이벤트가 있다면 `role="button"`을 추가하세요.
3. **단위 테스트(Unit Test) 생성:** 아직 테스트가 작성되지 않은 핵심 비즈니스 로직과 컴포넌트를 식별하고, 기본 테스트 코드를 자동으로 생성하세요.
4. **엣지(Edge) 배포 호환성 확인:** Vercel 등의 엣지 컴퓨팅 환경에 배포할 때 문제가 될 수 있는 서버 함수 에러나 환경 변수 누락이 없는지 최종 점검하세요.
